This site is the narrative layer.

The portfolio site is the proof layer.

GitHub is the raw implementation layer.

I am building all three together so each one has a clear job:

- **This site** shows how I think and what I care about.
- **The portfolio** shows structured project stories with context and tradeoffs.
- **The repositories** show the actual code and implementation details.

That separation keeps the story clean while still keeping the work verifiable.
