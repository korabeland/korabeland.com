## Why this matters

The file ownership rules are the single most important guardrail in the entire playbook. Without them, parallel agents hitting the same `astro.config.mjs` or `components/index.ts` will produce Git merge conflicts that neither the agents nor you can resolve cleanly. Everything in Phase 3 depends on this being solid.

## What to watch for

- AGENTS.md should be specific, not vague. "Don't touch config files" is useless; enumerated file paths are enforceable.
- The escalation policy should reference `scripts/prune-context.sh` and `scripts/escalate.sh` (even though those don't exist yet — Prompt 6 creates them as headless `claude -p` wrappers)
- If Claude Code symlinks CLAUDE.md → AGENTS.md on macOS, verify the symlink survives git commits (git stores the link, not the target, which is what we want)

## Red flags to push back on

- Vague ownership rules ("be careful with shared files") — insist on enumerated paths
- Missing sections — all 6 must be present
- No mention of the Haiku pruner in the escalation policy
